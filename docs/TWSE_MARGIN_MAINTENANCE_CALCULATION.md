# 上市大盤融資維持率自算方式

## 目的

`public/twse-margin-balance.html` 不再依賴 MacroMicro 的瀏覽器擷取資料，而是使用證交所公開資料固定產生上市大盤融資維持率。

目前採用的是早期跨日期試算後的實證公式，不是證交所公布的正式指標公式。此公式在 2026/07/23 已出現 4.6315 個百分點的明顯誤差，不應再把「排除全部註記 X」視為已驗證規則。若未來取得更完整的官方定義，應提高 `formulaVersion` 並保留舊資料，避免歷史數值在沒有紀錄的情況下被改寫。

2026/07/20、07/21、07/23 的 X 名單生命週期、除權息與多組候選算法，詳見：

- [大盤融資維持率 X 名單與除權息實驗紀錄](./TWSE_MARGIN_MAINTENANCE_X_EXPERIMENTS.md)

## 固定公式

```text
大盤融資維持率
=（TWSE STOCK 融資股票市值
  − 當日處置股融資股票市值
  − 註記 X 股票融資股票市值
  − KY 股票融資股票市值）
  ÷ 證交所融資金額今日餘額
  × 100%
```

逐檔市值：

```text
融資股票市值 = 融資今日餘額（張）× 1,000 × 當日收盤價
```

腳本會先對 `selectType=STOCK` 回傳的每一檔股票逐檔計算，再把所有個股市值加總成 `baseline.stockMarketValue`，不是用大盤總張數乘平均股價估算。任何仍有融資張數但缺少收盤價的股票都會列入 `dataQuality.missingPrices`；這些缺價目前以 0 計入，仍會使基礎分子低估。

分母使用證交所「信用交易統計」中的 `融資金額(仟元)／今日餘額 × 1,000`，不因 ETF、處置股、註記 X 或 KY 股票而扣減。這個官方金額代表歷史融資借款本金的逐日遞推結果：

```text
今日融資餘額
= 昨日融資餘額
  + 今日融資買進金額
  − 今日融資賣出金額
  − 今日融資現金償還金額
```

買進及賣出金額取決於各筆融資交易價格、張數與融資成數，因此不能用「當日收盤價 × 當日融資餘額張數 × 融資成數」回推替代。後者是每日盯市估值，不是 TWSE 官方歷史成本遞推後的借款本金，行情與持倉成本改變時可能產生很大的時間差匯總誤差。

公式版本：`twse-stock-exclude-punish-short-sale-restricted-ky-v1`

## 排除條件

### 股票集合

使用證交所融資融券餘額查詢的 `selectType=STOCK`：

```text
https://www.twse.com.tw/rwd/zh/marginTrading/MI_MARGN
```

`STOCK` 會排除 ETF，但仍包含存託憑證。實測 2026/07/21 存託憑證融資市值只影響約 0.1135 個百分點，排除後對貼近 MacroMicro 幫助很小，因此目前不另行排除。

### 處置股

使用證交所當日處置股查詢：

```text
https://www.twse.com.tw/rwd/zh/announcement/punish
```

查詢參數使用相同的 `startDate`、`endDate`，並設定 `querytype=3`。

### 註記 X

使用 `selectType=STOCK` 回傳資料最後一欄 `註記`。凡註記文字包含 `X` 的股票皆排除。

公開註記說明中，`X` 代表停止融券；目前採用它是因為跨 2026/07/20、2026/07/21 的實證結果最貼近 MacroMicro，並不代表證交所正式定義大盤融資維持率必須排除停止融券股票。

### KY 股票

凡證券名稱包含 `-KY` 的股票皆排除。這是目前最能解釋 2026/07/21 差異的排除集合之一；但仍屬實證規則。

## 交叉試算結果

以 `selectType=STOCK` 股票融資市值為基礎，再排除處置股、註記 X 及 KY 股票：

| 日期 | 自算結果 | 比對值 | 差距 |
| --- | ---: | ---: | ---: |
| 2026/07/20 | 143.1311% | 143.4932% | -0.3621 個百分點 |
| 2026/07/21 | 150.7991% | 150.7912% | +0.0079 個百分點 |
| 2026/07/23 | 151.0767% | 155.7082% | -4.6315 個百分點 |

07/21 的高度吻合無法證明「排除全部 X」正確。07/23 當天新增的 22 檔 X 股票市值約 236.10 億元，若全部排除，單日約降低 4.05 個百分點，是誤差突然放大的主要來源。

這個公式取代舊的 `exclude-etf-punish-notice-clause-11-v1`，主因是舊公式在 2026/07/21 失效：

| 日期 | 舊公式結果 | 比對值 | 差距 |
| --- | ---: | ---: | ---: |
| 2026/07/20 | 143.1670% | 143.4932% | -0.3262 個百分點 |
| 2026/07/21 | 160.5474% | 150.7912% | +9.7562 個百分點 |

2026/07/21 的新公式排除市值拆解：

```text
STOCK 融資股票市值：961.149B
目標分子：869.306B
需排除市值：約 91.843B

當日處置：7.931B
註記 X：20.768B
KY：66.746B
重疊後排除合計：91.798B
```

需要更多 MacroMicro 歷史日期驗證，尤其是 KY、註記 X 是否只是短期市場狀態下的近似規則。現階段已確認 X 與除權息時程高度相關，不能只因欄位出現 X 就無條件排除。

## 產生方式

執行：

```bash
node scripts/calculate_twse_margin_maintenance.js --date YYYYMMDD
```

必要的本機資料：

- `data_twse_mi_index/YYYYMMDD_twse_mi_index.json`

腳本另外即時取得證交所的融資金額統計、`selectType=STOCK` 融資餘額及處置股資料，並輸出：

```text
data_twse_margin_maintenance/YYYYMMDD_twse_margin_maintenance.json
```

輸出會保存分子、分母、公式版本、排除股票及各股票的融資張數、收盤價和市值，方便日後稽核。

## GitHub Actions

`.github/workflows/calculate-twse-margin-maintenance.yml` 會在台北時間平日 21:05、22:05、23:05 執行，也可以從 GitHub Actions 手動指定日期。workflow 會先確認：

- 同日融資餘額 CSV 已存在
- 同日 MI_INDEX 收盤價 JSON 已存在
- 同日融資維持率 JSON 尚未產生

全部條件成立時才執行：

```bash
node scripts/calculate_twse_margin_maintenance.js --date YYYYMMDD
```

缺少必要檔案或同日結果已存在時會正常略過，讓下一次排程再次檢查。新結果會提交至 `data_twse_margin_maintenance/`。

## 頁面顯示

`public/twse-margin-balance.html` 依日期選單載入同日的自算 JSON，顯示：

- 自算融資維持率日期
- 大盤融資維持率
- 前次已有計算資料的交易日維持率
- 較前次增減百分比

若所選日期沒有自算 JSON，頁面只顯示「尚未產生自算融資維持率」，不會退回使用 MacroMicro 數字。

## 已知限制

- 無收盤價但仍有融資餘額的股票，其市值目前按 0 計算，並列在輸出 JSON 的 `dataQuality.missingPrices`。
- 依 TWSE 操作辦法第 54 條，停止買賣或暫停交易應採停止或暫停前一營業日收盤價；其他無當日收盤價情形應依揭示買賣價與開盤競價基準決定。現行以 0 計算不符合精準估值要求，後續應修正。
- 不可把 `baseline.stockMarketValue × 融資成數` 當成官方分母；這會把盯市市值和歷史借款成本混為一談。
- 處置資料若證交所端點暫時失敗，腳本會停止且不輸出結果，避免把缺少排除名單的數字當成正式值。
- 註記 `X` 及 KY 排除仍是實證規則，不是官方公布的大盤融資維持率定義。
- 目前正式腳本仍排除全部註記 `X`；07/23 已證明此規則可能大幅低估，修改正式公式前應先以更多歷史日期驗證候選規則。
- 玩股網資料只有除權息日及股利資料，沒有現金股利實際發放日，因此無法直接驗證「除息後加回未發放股利」假設。TWSE 第 53 條可確認的是除權息日前六個營業日扣除權息值，兩種規則不可混為一談。
