# 大盤融資維持率 X 名單與除權息實驗紀錄

## 文件目的

這份文件保存 2026/07/20、2026/07/21、2026/07/23 對照 MacroMicro 後的觀察、已測試算法及尚待驗證的假設，避免下次調整 `scripts/calculate_twse_margin_maintenance.js` 時重做相同實驗。

這是研究紀錄，不是已確認的官方公式。截至 2026/07/24，正式腳本仍採用：

```text
（TWSE STOCK 融資股票市值
  − 當日處置股融資股票市值
  − 全部註記 X 股票融資股票市值
  − KY 股票融資股票市值）
÷ 證交所融資金額今日餘額

```

## 已確認的計算邊界

這次研究先固定以下邊界，避免把不同問題混在一起：

1. 分子使用每檔「融資餘額張數 × 1,000 × 當日收盤價」推算融資股票市值。
2. 分母使用 TWSE 公布的「融資金額今日餘額」，它是原始借款金額的加總；不把融資張數再乘 60% 當成分母，也不把分子乘融資成數。
3. 本研究只算 TWSE 集中市場，不混入 TPEx 上櫃資料。
4. MacroMicro 公開說明只明確表示不含 ETF，沒有公開 X、KY、處置股及除權息應收款的完整細節；這些都必須用跨日期資料反推。
5. 官方假除權息處理和「現金股利發放前加回應收款」是兩個不同假設，實驗時應分開計算，不能合併後就把差額歸因給其中一項。

TWSE `TWT48U` JSON 適合取得近期除權息預告及欄位定義，但 2026/07/24 實測只回傳 2026/07/23 至 2026/08/05 的近期 161 筆資料，加入 `date`、`startDate` 或 `endDate` 也沒有取得 07/21 歷史區間。因此這次 2025～2026 歷史交叉比對改用本機玩股網 JSON；未來若 TWSE 端點行為改變，應優先重新測試官方歷史查詢。

### 分子必須逐檔計算

不能用市場融資總張數、平均股價或其他彙總欄位近似分子。正確的基礎分子是：

```text
Σ（每檔融資今日餘額張數 × 1,000 × 該檔當日收盤價）
```

目前 `calculate_twse_margin_maintenance.js` 已按這個方式執行：

1. 讀取 TWSE `selectType=STOCK` 的每檔融資今日餘額。
2. 以證券代號對應同日 `MI_INDEX` 收盤價。
3. 逐檔產生 `marketValue`。
4. 用 `records.reduce(...)` 加總成 `baseline.stockMarketValue`。
5. 最後才依實證規則扣除處置股、X 與 KY。

07/23 共處理 1,059 筆股票融資紀錄，逐檔加總的基礎市值是 998,403,124,140 元。當天 1459 聯發仍有 437 張融資、1589 永冠-KY 仍有 793 張融資，但缺少收盤價，現在以 0 計入並列在 `dataQuality.missingPrices`。要追求精準分子時，缺價不應靜默略過，應補價或讓計算失敗；不過這兩檔不是 4.6315 點落差的主要來源。

### 官方分母是歷史成本遞推，不是收盤價回推

TWSE 公布的融資金額今日餘額可理解為：

```text
今日融資餘額
= 昨日融資餘額
  + 今日融資買進金額
  − 今日融資賣出金額
  − 今日融資現金償還金額
```

其中融資買進及賣出金額依實際交易價格、張數與融資成數計算。經過持續買進、賣出及現金償還後，官方分母反映的是尚未償還借款本金的歷史成本組成。

不能改用：

```text
當日收盤價 × 當日融資餘額張數 × 融資成數
```

這個算法是用當日市價回推借款，會把歷史成本分母改成每日盯市分母；股價和融資持倉成本偏離時，兩者可能出現嚴重差距。有些財經網站可能自行使用回推值，所以比對前要先確認雙方分母口徑。

目前腳本的 `getFinancingAmount()` 直接讀取 TWSE `selectType=ALL` 中「融資金額」的「今日餘額」並乘 1,000，採用的是官方分母，沒有用收盤價回推。

TWSE 官方分母：582,626,351,000 元

## 其他四項常見落差的核對

### 1. 除權息後加回尚未發放現金股利

「除息後把每股現金股利乘融資股數，加回分子直到發放日」仍是值得測試的 MacroMicro 反推假設，但目前不能寫成已確認的 TWSE 市場彙總公式。

TWSE《證券商辦理有價證券買賣融資融券業務操作辦法》第 53 條明確規定的是：個別信用帳戶在除權息日前六個營業日，除現金增資外，融資擔保品市值以當日收盤價扣除息值或權值計算。這是「除權息日前預先扣除」的規則，不等同「除息後直到發放日前加回現金股利」。

官方規則：

```text
https://twse-regulation.twse.com.tw/TW/law/DOC01.aspx?FLCODE=FL007121&FLNO=53
```

因此後續比較至少應分開保存三種算法：

1. 純當日收盤價。
2. 依第 53 條，在除權息日前六個營業日扣除權息值。
3. 除息後加回尚未發放現金股利的網站反推假設。

目前玩股網檔案沒有現金股利發放日，第三種算法仍無法精準完成。先前 0、7、14、30、60 天固定回看實驗也顯示，現金股利加回本身不足以解釋 07/23 的差距。

### 2. ETF、TDR 與創新板範圍

2026/07/23 直接檢查 TWSE `selectType=STOCK` 的 1,059 筆明細：

| 類別 | 筆數 | 融資股票市值 | 對官方分母的影響 |
| --- | ---: | ---: | ---: |
| 00 開頭 ETF | 0 | 0 | 0 |
| TDR（91 開頭或 `-DR`） | 7 | 639,982,710 元 | 0.1098 點 |
| 創新板（`-創` 或 `KY創`） | 14 | 6,525,224,300 元 | 1.1200 點 |

結論：

- `selectType=STOCK` 已排除 ETF；`data_twse_margin_balance` 的全證券 CSV 雖然看得到 0050 等 ETF，但正式計算腳本沒有使用那個集合當分子。
- TDR 仍在 STOCK 集合中，但影響只有約 0.1098 點。
- 創新板確實在 STOCK 集合中，影響約 1.1200 點，值得保留成候選變因。
- 07/23 現行結果比 MacroMicro 低 4.6315 點；再排除 TDR 或創新板會讓結果更低，方向相反，不能單獨解釋本次落差。

「加權指數成分股」和「融資維持率股票集合」也不能直接視為同一件事。TWSE 加權指數成分股原則是所有上市普通股，另對新上市、停止買賣及變更交易方法股票訂有納入或移除時點；TDR 並非普通股。若要測試「只留加權指數成分股」，應建立每日成分股快照，不能只用代碼前綴猜測。

加權指數編製要點：

```text
https://twse-regulation.twse.com.tw/TW/law/DAT0201_print.aspx?FLCODE=FL047579
```

分母方面，TWSE `selectType=ALL` 只提供整體官方融資金額。若某候選算法要同時從分母剔除 TDR 或創新板，必須取得或重建那些股票的歷史未償還融資本金，不能拿「當日市值 × 60%」直接扣除。

### 3. 個股融資比率可能不是 60%

這個觀察成立，但要放在正確的位置：

- TWSE 可因價格波動、成交量異常或股權集中調低個別有價證券融資比率；部分處置情況甚至可調為 0。
- 不同融資比率會影響新增融資借款本金，因而逐步反映在 TWSE 官方總融資金額分母。
- 分子仍是融資股數的擔保品市值，不應再乘 60%、50% 或 40%。
- 目前腳本直接使用官方總分母，因此不需要為每檔融資比率重新加權分子。

官方規則可參考操作辦法第 26、33、34 條。若未來要從分母排除特定股票，才需要追蹤逐檔、逐日融資比率及歷史買賣成本；只靠當日融資張數無法把該股票的官方借款本金精準拆出來。

```text
https://twse-regulation.twse.com.tw/TW/law/DAT0202_print.aspx?FLCODE=FL007121&LCC=3&LCNOS=++26+++
https://twse-regulation.twse.com.tw/TW/law/DOC01_print.aspx?FLCODE=FL007121&FLNO=33
https://twse-regulation.twse.com.tw/TW/law/DOC01_print.aspx?FLCODE=FL007121&FLNO=34
```

### 4. 停止買賣、暫停交易及無當日收盤價

這是目前腳本已確認的資料品質缺口。操作辦法第 54 條規定：

- 停止買賣或暫停交易：使用停止或暫停前一營業日收盤價。
- 非停牌但當日無收盤價：依收盤時最高買進申報價、最低賣出申報價與開盤競價基準決定。

官方規則：

```text
https://twse-regulation.twse.com.tw/TW/law/DOC01_print.aspx?FLCODE=FL007121&FLNO=54
```

目前 `loadClosingPrices()` 只讀 `MI_INDEX` 的當日收盤價，找不到就設為 0。07/23 有：

| 代號 | 名稱 | 融資張數 | 已知情況 | 對現行結果的影響 |
| --- | --- | ---: | --- | ---: |
| 1459 | 聯發 | 437 | 07/22 收盤價 11.85 元 | 補回約 5,178,450 元，約 +0.0009 點 |
| 1589 | 永冠-KY | 793 | 07/23 MI_INDEX 無資料 | 現行公式已因 KY 排除，不影響現行分子 |

所以缺價處理必須修正，否則無法稱為精準分子；但它不是 07/23 低估 4.6315 點的主因。較安全的正式機制應是：

1. 先識別停止買賣或暫停交易狀態。
2. 停牌股票取停止前最後有效收盤價。
3. 其他無成交股票依第 54 條使用揭示買賣價與開盤競價基準。
4. 若仍無法取得合法估值，不產生正式結果，而不是以 0 靜默計入。

## X 欄位不能直接等同融資排除

TWSE 公開註記中，`X` 代表停止融券，不是停止融資。另一個時間陷阱是：每日融資融券餘額表的註記描述的是次一營業日狀態，但現行腳本用目標日的 X 名單直接計算目標日。

因此至少有兩個風險：

1. 業務含義不同：停止融券不必然代表融資餘額不應納入分子。
2. 日期可能錯位：當日 X、前一交易日 X，以及除權息前後的 X 生命週期可能代表不同計算狀態。

## 當日 X 與前一交易日 X 的實驗

名詞：

- `same`：排除目標日 X。
- `prev`：排除前一交易日 X。
- `intersection`：只排除兩日皆為 X。

結論：

- 可以取消 KY 規則重頭測試，且這個反證實驗已完成。
- 單靠目前掌握的除權息因素，無法取代 KY 排除造成的 10 點左右效果。
- 這不代表「排除全部 KY」就是正確的正式規則，只表示若取消它，必須找到另一個約 600～680 億元級別、且能跨日期成立的分子範圍或排除集合。
- 下一步應分析 KY 為何形成穩定的大額差異，例如網站是否只採特定市場／指數母體、第一上市公司是否另有處理，或 KY 是否與某個尚未辨識的信用交易狀態高度重疊；不應用數十億元以下的股利調整硬補。

## 目前最值得繼續驗證的候選規則

目前在三個對照日期上表現較穩定的經驗規則：

1. 排除處置股及 KY。
2. 對每檔 X 尋找目標日前 1 天至後 14 天內最近的玩股網除權息事件。
3. 有股票股利或現金增資配股的 `rights` 類 X：排除。
4. 只有現金股利的 `cash` 類 X：只有除權息日在目標日後 0 至 3 個日曆日內才排除。
5. 視窗內找不到事件的 `unmapped` 類 X：暫時排除。
6. 其他距離較遠的 `cash` 類 X：不排除。

試算結果：

| 日期 | 候選規則 | MacroMicro | 差距 |
| --- | ---: | ---: | ---: |
| 2026/07/20 | 143.5449% | 143.4932% | +0.0517 |
| 2026/07/21 | 150.9482% | 150.7912% | +0.1570 |
| 2026/07/23 | 155.3131% | 155.7082% | -0.3951 |

三日平均絕對誤差約 0.2013 個百分點，最大絕對誤差 0.3951。它比現行「排除全部 X」穩定，但只有三個日期，且參數是從這三日反推，存在過度擬合風險。

另一個值得保留的觀察是「排除 rights，以及除權息日在未來 5 日內的 cash，但不排除 unmapped」：

| 日期 | 結果 | 與 MacroMicro 的差距 |
| --- | ---: | ---: |
| 2026/07/21 | 151.3734% | +0.5822 |
| 2026/07/23 | 155.7787% | +0.0705 |

它在 07/23 更接近，但 07/21 誤差較大，因此尚不能判斷 3 日或 5 日視窗何者合理。

## 下次實驗建議

正式修改公式前，建議依序完成：

1. 建立獨立的算法比較腳本，不覆寫正式輸出。
2. 每個交易日同時計算 `same`、`prev`、`intersection`、`no X` 與事件分類候選規則。
3. 在輸出中保存每檔 X 的分類、最近事件日、與目標日相差天數、市值及影響百分點。
4. 補抓更多 MacroMicro 歷史日期，至少涵蓋非除權息旺季、除息密集日及 X 大量增減日。
5. 若要測試股利應收款，補充現金股利發放日，而不是固定假設 7、14、30 或 60 天。
6. 分開驗證 TWSE 上市與 TPEx 上櫃，避免市場資料混用。
7. 找到能跨日期成立的規則後，再提高 `formulaVersion`，保留舊版歷史結果及本文件的驗證表。

## 現階段結論

- 07/21 的近乎零誤差只是單日證據，不能證明全部 X 都應排除。
- 07/23 的 4.6315 點落差主要由新增的 22 檔 X、約 236.10 億元市值造成。
- X 與除權息生命週期高度相關，是否排除應至少考慮事件種類及日期距離。
- 單純改用前一交易日 X、完全不排除 X，或加回固定期間現金股利，都無法同時解釋三個日期。
- 「rights、近期 cash、unmapped 分類處理」是目前較值得擴大驗證的候選規則，但尚不應直接取代正式公式。
