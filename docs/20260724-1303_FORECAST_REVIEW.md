# 20260724-1303 預測報告審查與改進要求

## 1. 文件目的

本文件審查以下預測報告：

```text
public/predictions/20260724-1303.html
```

審查目標不是事後判斷預測方向是否命中，而是確認：

1. 報告是否只使用資訊截止時間以前的資料。
2. 所有數字是否能由專案內原始資料重算。
3. 計分是否符合 `docs/STOCK_FORECAST_METHODOLOGY.md`。
4. 回測結果是否有可重現的程式、樣本與輸出。
5. 報告是否清楚區分「資料完整度」與「預測可信度」。

另一個模型修改報告時，應逐項處理本文件的問題，不能只改文字或排版。

---

## 2. 審查結論

目前報告的核心方向「偏空」可以由 2026-07-23 的資料重算成立，方向分數 `-9` 也可以成立。

但是，這份報告目前不適合被視為可信的事前預測紀錄，主要原因如下：

1. 回測統計無法在專案中追溯或重現。
2. 報告在預測交易日收盤後才 commit，無法證明是事前預測。
3. 計分表的文字與方法規格不一致。
4. 成交量與主力合計有錯，分點反轉比例的呈現範圍不清楚。
5. 情境文字與情境價格區間互相矛盾。
6. 「100%・高信心」容易被誤解成預測正確率。

建議保留可重算的價量與籌碼分析，撤下或重新完成無法驗證的回測，並將報告性質改成「歷史回推／無前視模擬」。

## 4. 方向分數重算

依 `docs/STOCK_FORECAST_METHODOLOGY.md` 第 10 節：

| 分類 | 條件 | 分數 |
|---|---|---:|
| 價格 | `r1 <= -3%` | -1 |
| 趨勢 | `close > SMA20` | +1 |
| 當日走勢 | `intraday_return <= -3%` | -1 |
| 成交量 | 下跌但 `volume_ratio < 1.2` | 0 |
| 相對強度 | `relative_strength <= -3%` | -2 |
| 三日報酬 | `r3` 未達正負 8% | 0 |
| 法人 | `institutional_ratio <= -10%` | -2 |
| 五日法人 | `institutional_5d_ratio <= -5%` | -1 |
| 法人同向 | 外資負、投信正，不同向 | 0 |
| 主力 | `main_net_ratio <= -5%` | -2 |
| 主力反轉 | 前日買超主力轉賣且占量至少 5% | -1 |
| 融資 | 變化率未達正負 1% | 0 |
| **合計** |  | **-9** |

`-9` 對應「偏空」，所以原始方向與最終方向可以保留。

這只能證明核心輸入與方向分數可以由 7/23 或更早的資料重算，不能單獨證明報告製作過程完全沒有查看 7/24 結果。由於報告是在 7/24 收盤後 commit，時間戳仍不足以支持「事前預測」或「已證明零偷看」的說法。

### 必須修正的計分矛盾

原報告寫：

```text
外資與自營商同向賣超 (-1)
```

但方法規格的條件是：


若報告真的把「外資與自營商同為負」計入 `-1`，總分就不會是 `-9`。修改後必須提供完整逐項計分表，避免顯示內容與實際計算不同。

## 5. ATR 與價格區間核對

依方法規格，以最近 14 個交易日 TR 的簡單平均計算：

```text
ATR14 = 18.8571 元
```

偏空中心：

```text
center = 197 - 0.35 × 18.8571
       = 190.40
```

依合法跳動單位顯示為約 `190.50 元`，可以接受。

可能收盤區間：

```text
190.40 ± 0.5 × 18.8571
= 180.97 ～ 199.83
```

依跳動單位顯示約為 `181.00～200.00 元`，可以接受。

可能盤中區間：

```text
190.40 ± 18.8571
= 171.54 ～ 209.26
```

報告寫 `171.50～209.00 元`。上緣應依專案統一的跳動單位取整函式重新產生，不能由文字模型自行判斷。若採最接近 0.5 元，應為約 `209.50 元`。

## 6. 必須修正的重大問題

### P0：回測統計無法追溯

原報告宣稱：

| 指標 | 報告數值 |
|---|---:|
| 樣本數 | 142 |
| 隔日續跌或震盪下探比例 | 68.3% |
| 隔日平均報酬率 | -1.42% |
| 法人大賣疊加後下探率 | 74.1% |

目前專案中找不到：

1. 回測程式。
2. 回測輸入期間。
3. 142 筆樣本的日期與股票代號。
4. 每筆樣本的特徵與隔日報酬。
5. 回測輸出 JSON。
6. 中位數、第 10 百分位、第 90 百分位。

此外，報告使用的條件：

```text
單日跌幅 >= 3%
開盤至收盤跌幅 >= 3%
相對強度 <= -3%
法人賣超占成交量 >= 10%
```

並不是方法文件第 13.2 節的固定回測群組。

#### 改進要求

只能選擇以下其中一種處理方式：

1. 完成可重現回測，加入 script、樣本明細 JSON、統計輸出 JSON，再保留數字。
2. 在完成回測以前，移除 142、68.3%、-1.42%、74.1% 等全部統計。

不得只在文字後面增加「僅供參考」而繼續保留無法重現的數字。

### P0：報告不是可驗證的事前預測

報告預測交易日是 2026-07-24，但加入報告的 commit 為：

```text
commit: 32e60cec6dee7006d3509ac570e69eb3e451ca02
author time: 2026-07-24T18:31:27+08:00
commit time: 2026-07-24T18:31:34+08:00
```

建立時間晚於 7/24 收盤，因此不能被視為有時間戳佐證的事前預測。

#### 改進要求

將本頁明確改標為：

```text
以 2026-07-23 為資訊截止點的歷史回推／無前視模擬
```

並說明：

```text
本頁在 2026-07-24 收盤後建立，不能作為真實事前預測績效紀錄。
```

未來正式預測應在預測交易日開盤以前 commit，並保存：

1. `generated_at`
2. `forecast_date`
3. `base_trade_date`
4. `information_cutoff`
5. 使用資料檔案及其內容雜湊
6. 方法版本或 commit SHA

### P1：資料截止時間聲明缺乏證據

原報告寫「2026-07-23 15:30 已完整取得法人、融資與券商分點」。

專案資料只能證明資料日期是 7/23，不能證明所有資料在 15:30:00 已發布並被取得。

#### 改進要求

若沒有每個來源的實際抓取時間，應改成：

```text
核心資料交易日：2026-07-23
報告計算僅使用標記為 2026-07-23 或更早的資料。
```

不得宣稱精確到 15:30:00 已取得全部資料，除非保存抓取時間並驗證來源當時已發布。

### P1：「放量長黑」敘述錯誤

`volume_ratio = 1.0063`，不符合 `>= 1.2` 的放量條件。

#### 改進要求

改成：

```text
接近五日均量的長黑 K，並伴隨法人與主要分點賣壓。
```

同時確認方向分數沒有誤扣成交量的 `-1`。

### P1：情境文字與價格區間矛盾

空方情境文字說回測 `185.00～187.85 元` 支撐，但醒目價格顯示 `171.50～181.00 元`，兩者沒有重疊。

#### 改進要求

每個情境必須分開顯示：

1. 預期收盤區間。
2. 可能盤中極端區間。
3. 關鍵支撐／壓力。

不得把「可能盤中區間下半部」直接當作空方收盤區間。

## 7. 信心標示問題

原報告顯示：

```text
100% · 高信心
```

方法文件中的 100% 是資料完整度權重加總，不是：

1. 預測正確率。
2. 上下方向命中率。
3. 回測勝率。
4. 價格區間覆蓋率。

在回測尚未能重現的情況下，「高信心」尤其容易誤導。

原報告使用「100%・高信心」並不是違反方法規格；它確實依照方法文件第 12 節，把資料完整度 90～100 對應為高信心。本項屬於讀者介面與名詞定義需要澄清，不是核心算式錯誤。

### 改進要求

建議拆成兩個欄位：

```text
資料完整度：100%
模型證據強度：中等（回測尚未驗證）
```

如果仍要沿用方法文件的「高信心」名稱，至少必須明示：

```text
此處高信心只表示核心資料完整，不代表預測命中率。
```

## 11. 審查限制

本次已完成 HTML 原始碼、專案資料與方法規格的靜態核對。HTML 語法檢查沒有回報明顯錯誤。

本次嘗試使用瀏覽器進行實際視覺檢查時，瀏覽器執行環境發生初始化錯誤，因此尚未完成桌面與手機尺寸的截圖層級視覺 QA。另一個模型修改後，仍應實際開啟頁面檢查：

1. 桌面版三欄情境是否正常。
2. 手機版表格是否可閱讀。
3. 長路徑與數字是否溢出。
4. 顏色是否錯誤暗示漲跌方向。
5. 首頁連結是否能正確開啟報告。
